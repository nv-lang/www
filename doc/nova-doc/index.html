Quick start

nova doc reads doc-comments from .nv source files and renders them as Markdown, JSON, or HTML. It also runs doc-tests embedded in code examples.

nova doc path/to/file.nv                # Markdown to stdout
nova doc path/to/file.nv --format json  # JSON to stdout
nova doc path/to/file.nv --test         # Run doc-tests
nova doc path/to/file.nv --check        # Validate docs (CI)

Pass a directory to operate on an entire workspace. By default only exported items appear in output; add --include-private to include private items too.

Doc-comments

Two prefixes are recognised:

  • /// outer — attaches to the declaration that follows it
  • //! inner — attaches to the enclosing module (placed at the top of the file)

Four or more slashes (////) are a plain comment, not a doc-comment.

//! Module-level doc — describes the whole file.

module my.module

/// Returns twice the value of `x`.
///
/// Long description goes here. It can span
/// multiple lines.
///
/// # Examples
///
/// ```nova
/// let x = double(3)
/// assert(x == 6)
/// ```
export fn double(x int) -> int => x * 2

The first sentence (ending with .) becomes the summary. Everything after the first blank line is the extended description.

Sections

Headings of the form # Name inside a doc-comment mark standard sections. Recognised names, in the recommended order:

Section Purpose
# Examples Usage examples. ```nova blocks are doc-tests.
# Errors Which Fail[X] errors the function may return.
# Panics Conditions under which the function panics at runtime.
# Safety Invariants callers must uphold when using unsafe APIs.
# Effects Lists effects if not obvious from the signature.
# Contracts Pre/post-conditions (see Spec §5).
# Since Version in which the item first appeared.
# See also Cross-references to related items.
# Deprecated Reason for deprecation and recommended replacement.

Any other # Heading is preserved as free-form text within the doc.

/// Divides `a` by `b`.
///
/// Returns the integer quotient.
///
/// # Errors
///
/// Returns `DivisionByZero` if `b == 0`.
///
/// # Panics
///
/// Never panics.
///
/// # Since
///
/// 0.1.0
///
/// #stable
export fn divide(a int, b int) -> int =>
    a / b

Intra-doc links

Reference other items with square brackets:

/// Returns a [Point] for the given coordinate.
/// See also [translate] and [std.math.abs].

Resolution rules:

  • [Name] — short match by item name
  • [Type.method] — method form
  • [mod.path.Name] — fully-qualified path

Broken links produce a warning from nova doc --check and appear as "target_id": null in JSON output.

Doc-tests

Every ```nova block in a doc-comment is a doc-test — compiled and run by nova doc --test.

Modifiers

Append comma-separated modifiers to the opening fence:

Modifier Behaviour
no_run Compile but do not execute.
ignore Skip compilation entirely (display only).
compile_fail Expect a compile error — test passes if compilation fails.
should_panic Expect a runtime panic.
must_verify Expect successful SMT verification (requires contracts).

Modifiers can be combined: ```nova,no_run,ignore.

Hidden lines

Lines beginning with # are hidden in rendered output but included in compilation. Use them for imports and test setup:

/// ```nova
/// # import std.io
/// let r = compute()
/// assert(r == 42)
/// ```

Stability and deprecation

Stability is declared with doc-attributes — single-line annotations placed between the doc-comment and the item. The syntax is #name(...), not Rust-style #[...]:

/// Stable function, available since 1.0.
#stable(since = "1.0.0")
export fn add(a int, b int) -> int => a + b

/// Experimental API — may change.
#experimental(note = "may change in 0.5")
#since("0.3.0")
export fn experimental_api() -> int => 0

/// Old function — use `add` instead.
#deprecated("Use `add` instead.")
#since("0.1.0")
export fn plus(a int, b int) -> int => a + b

Supported attributes (D105):

Attribute Parameters
#deprecated("note") Reason and recommended replacement
#since("X.Y") Version the item first appeared
#stable / #stable(since = "X") Stable public API
#unstable(feature = "name") Behind a named feature flag
#experimental(note = "..."?) Proof-of-concept, may be removed
#hide_doc Item is exported but hidden from docs
#doc_alias("alt", ...) Search aliases
#doc(inline) / #doc(no_inline) Re-export rendering behaviour
#doc(summary = "...") Override the auto-extracted summary

CLI flags

Flag Description
--format markdown|json Output format (default: markdown)
--include-private Include non-export items
--test Run doc-tests instead of rendering
--check Validate docs — broken links, missing summaries; exits 1 on issues
--json-schema Print the embedded JSON Schema 2020-12 (D107)

JSON output

Pass --format json to get machine-readable output conforming to schema v1 (D107). Output is byte-for-byte deterministic — suitable for diffs and reproducible builds.

Top-level fields: format_version, nova_version, modules[], items[], links[], doc_tests[]. Run nova doc --json-schema for the full JSON Schema.

Common jq recipes:

# All exported functions with their summary
nova doc src/api.nv --format json \
  | jq '[.items[] | select(.kind=="fn" and .visibility=="export") | {id,summary}]'

# Items without a summary (undocumented)
nova doc src/api.nv --format json \
  | jq '[.items[] | select(.summary == null) | .id]'

# Broken intra-doc links
nova doc src/api.nv --format json \
  | jq '[.links[] | select(.target_id == null) | .text]'

# Unstable and experimental items
nova doc src/api.nv --format json \
  | jq '[.items[] | select(.stability.tier != null and .stability.tier != "stable") | {id, tier: .stability.tier}]'

CI integration

# Validate docs (broken links + missing summaries) — exits 1 on issues
nova doc src/api.nv --check

# Run doc-tests — exits 1 on any failure
nova doc src/api.nv --test

# Reproducible builds (suppresses the generated_at timestamp)
SOURCE_DATE_EPOCH=1700000000 nova doc src/api.nv --format json > api.json

Style guide

  1. First sentence is the summary — end it with ..
  2. Use imperative mood: "Returns X", not "This function returns X".
  3. Follow the section order in the Sections table above.
  4. Markdown subset: CommonMark + ```nova fenced blocks.
  5. # Examples is required for all exported functions.
  6. Deprecation notes must include the replacement and a #since attribute.
  7. Stability tiers: #stable / #unstable / #experimental.
