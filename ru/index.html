Nova

Язык общего назначения вокруг алгебраических эффектов, статических контрактов и AI-first эргономики.

Статус: Bootstrap. Nova в активной ранней разработке. Компилятор работает, стандартная библиотека частична, breaking changes происходят каждую неделю. Не использовать в продакшене.

Чем отличается

  • Эффекты в сигнатурах. Сеть, ввод-вывод, время, случайность, мутация — каждый побочный эффект объявлен в типе. Никакого скрытого control flow, никаких неожиданных await.
  • Контракты как first-class. requires / ensures / invariant проверяются статически (SMT) где возможно, в runtime в debug, zero-cost в release.
  • AI-first эргономика. Спроектирован под эпоху, где LLM пишут 50–80% кода: явные сигнатуры, локальность, ошибки компилятора как обучающий сигнал.
  • M:N runtime. Go-style fibers с work-stealing scheduler. Managed по умолчанию (Boehm GC сейчас, concurrent GC планируется), realtime nogc для hot path.

На вкус

fn fetch_user(id u64) Result[User, NotFound]
    requires id > 0
    ensures result is Ok(u) implies u.id == id
    uses Db, Log
{
    log("fetching user", id)
    match db.find_user(id) {
        Some(u) -> Ok(u)
        None    -> Err(NotFound)
    }
}

Эффекты (Db, Log) и постусловие прямо в сигнатуре — вызывающий код знает ровно что функция делает и что ожидать на выходе.
