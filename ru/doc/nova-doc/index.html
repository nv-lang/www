Быстрый старт

nova doc читает doc-комментарии из .nv-файлов и рендерит их в Markdown, JSON или HTML. Также запускает doc-тесты, встроенные в примеры кода.

nova doc path/to/file.nv                # Markdown в stdout
nova doc path/to/file.nv --format json  # JSON в stdout
nova doc path/to/file.nv --test         # Запустить doc-тесты
nova doc path/to/file.nv --check        # Проверить doc на ошибки (CI)

Можно передать директорию — тогда команда работает со всем workspace. По умолчанию в вывод попадают только export-элементы; --include-private включает и приватные.

Doc-комментарии

Распознаются два вида:

  • /// outer — относится к следующей за ним декларации
  • //! inner — относится к целому модулю (ставится в начале файла)

Четыре и более слеша (////) — обычный комментарий, не doc.

//! Модульный doc — описание всего файла.

module my.module

/// Возвращает удвоенное значение `x`.
///
/// Длинное описание. Может занимать
/// несколько строк.
///
/// # Examples
///
/// ```nova
/// let x = double(3)
/// assert(x == 6)
/// ```
export fn double(x int) -> int => x * 2

Первое предложение (до .) становится summary. Всё после первой пустой строки — расширенное описание.

Секции

Заголовки вида # Name внутри doc-комментария задают стандартные секции. Распознаваемые имена в рекомендуемом порядке:

Секция Назначение
# Examples Примеры использования. Блоки ```nova — doc-тесты.
# Errors Какие Fail[X]-ошибки может вернуть функция.
# Panics Условия, при которых функция паникует в runtime.
# Safety Инварианты для вызывающих unsafe-API.
# Effects Перечень эффектов, если они не очевидны из сигнатуры.
# Contracts Пред- и постусловия (см. Spec §5).
# Since Версия, с которой элемент появился.
# See also Ссылки на связанные элементы.
# Deprecated Причина устаревания и рекомендуемая замена.

Любой другой # Заголовок сохраняется как свободный текст в doc.

/// Делит `a` на `b`.
///
/// Возвращает целочисленный quotient.
///
/// # Errors
///
/// Возвращает `DivisionByZero`, если `b == 0`.
///
/// # Panics
///
/// Никогда не паникует.
///
/// # Since
///
/// 0.1.0
///
/// #stable
export fn divide(a int, b int) -> int =>
    a / b

Внутренние ссылки

Ссылки на другие элементы — в квадратных скобках:

/// Возвращает [Point] для переданной координаты.
/// См. также [translate] и [std.math.abs].

Правила резолвинга:

  • [Name] — короткий поиск по имени
  • [Type.method] — форма метода
  • [mod.path.Name] — полный путь

Сломанные ссылки выдают предупреждение в nova doc --check и появляются как "target_id": null в JSON-выводе.

Doc-тесты

Каждый блок ```nova в doc-комментарии — это doc-тест, который компилируется и запускается командой nova doc --test.

Модификаторы

Добавляются через запятую к открывающему ограничителю:

Модификатор Поведение
no_run Компилируется, но не запускается.
ignore Пропускается полностью (только отображение).
compile_fail Ожидается ошибка компиляции — тест проходит, если компиляция падает.
should_panic Ожидается runtime panic.
must_verify Ожидается успешная SMT-верификация (требует контрактов).

Модификаторы можно комбинировать: ```nova,no_run,ignore.

Скрытые строки

Строки, начинающиеся с # , скрыты в визуальном выводе, но включаются в компиляцию. Используются для импортов и setup-кода:

/// ```nova
/// # import std.io
/// let r = compute()
/// assert(r == 42)
/// ```

Стабильность и устаревание

Стабильность объявляется через doc-атрибуты — однострочные аннотации между doc-комментарием и элементом. Синтаксис: #name(...), не Rust-style #[...]:

/// Стабильная функция, доступна с 1.0.
#stable(since = "1.0.0")
export fn add(a int, b int) -> int => a + b

/// Экспериментальное API — может измениться.
#experimental(note = "may change in 0.5")
#since("0.3.0")
export fn experimental_api() -> int => 0

/// Устаревшая функция — используйте `add`.
#deprecated("Use `add` instead.")
#since("0.1.0")
export fn plus(a int, b int) -> int => a + b

Поддерживаемые атрибуты (D105):

Атрибут Параметры
#deprecated("note") Причина устаревания и рекомендуемая замена
#since("X.Y") Версия появления элемента
#stable / #stable(since = "X") Стабильное публичное API
#unstable(feature = "name") За named feature-флагом
#experimental(note = "..."?) Proof-of-concept, может быть удалено
#hide_doc Элемент экспортирован, но скрыт из docs
#doc_alias("alt", ...) Поисковые алиасы
#doc(inline) / #doc(no_inline) Рендеринг re-export'ов
#doc(summary = "...") Переопределить авто-summary

Флаги CLI

Флаг Описание
--format markdown|json Формат вывода (по умолчанию: markdown)
--include-private Включить non-export-элементы
--test Запустить doc-тесты вместо рендеринга
--check Проверить doc (сломанные ссылки, отсутствующие summary); exit 1 при ошибках
--json-schema Вывести встроенный JSON Schema 2020-12 (D107)

JSON-вывод

Флаг --format json выдаёт машиночитаемый вывод по схеме v1 (D107). Вывод детерминирован побайтово — подходит для диффов и воспроизводимых сборок.

Top-level поля: format_version, nova_version, modules[], items[], links[], doc_tests[]. Полная JSON Schema: nova doc --json-schema.

Распространённые jq-запросы:

# Все export-функции с summary
nova doc src/api.nv --format json \
  | jq '[.items[] | select(.kind=="fn" and .visibility=="export") | {id,summary}]'

# Элементы без summary (незадокументированные)
nova doc src/api.nv --format json \
  | jq '[.items[] | select(.summary == null) | .id]'

# Сломанные intra-doc ссылки
nova doc src/api.nv --format json \
  | jq '[.links[] | select(.target_id == null) | .text]'

# Unstable и experimental элементы
nova doc src/api.nv --format json \
  | jq '[.items[] | select(.stability.tier != null and .stability.tier != "stable") | {id, tier: .stability.tier}]'

CI-интеграция

# Проверить doc (сломанные ссылки + отсутствующие summary) — exit 1 при ошибках
nova doc src/api.nv --check

# Запустить doc-тесты — exit 1 при провале
nova doc src/api.nv --test

# Воспроизводимая сборка (подавить generated_at timestamp)
SOURCE_DATE_EPOCH=1700000000 nova doc src/api.nv --format json > api.json

Руководство по стилю

  1. Первое предложение — summary, заканчивается ..
  2. Повелительное наклонение: «Возвращает X», не «Эта функция возвращает X».
  3. Соблюдать порядок секций из таблицы Секции.
  4. Markdown-подмножество: CommonMark + ```nova-блоки.
  5. # Examples обязателен для всех export-функций.
  6. Deprecation note должен содержать замену и атрибут #since.
  7. Уровни стабильности: #stable / #unstable / #experimental.
